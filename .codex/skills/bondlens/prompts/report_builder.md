# BondLens Report Builder

生成 9 层结构的 BondLens 分析报告。**Layer -1（关系行动卡）是报告第一屏**，用户不读后面的长报告也能知道本周该做什么。

## 报告结构

### Layer -1: 关系行动卡（第一屏，必须）

参考 `prompts/action_brief_builder.md`，生成压缩版行动卡。

这是报告的**默认第一屏**。用户打开报告时最先看到这个。必须在 1 分钟内回答"我现在该怎么做"。

内容固定为：
- 当前局势（一句话判断 + 证据 + 置信度）
- 当前策略（低压稳定 / 适度推进 / 冲突修复 + 为什么）
- 本周 3 个动作（每个有触发条件、具体动作、证据 ID）
- 不要做的 5 件事（每个有原因和证据）
- 已经踩过的雷（事件→表现→后果→替代做法）
- 往后观察信号（信号→解读→应对）
- 可直接发的话（稳妥版/降压版/更直接版）

**质量标准**：每条建议必须有触发条件、具体动作、证据 ID、禁忌边界。不允许"保持适度联系"这种泛建议。

### Layer 0: 核心互动规则

从数据中提炼 3-5 条最重要的互动规则。

格式：
```
规则名称：{name}
证据：{IDs}
置信度：{level}
描述：{description}
```

### Layer 1: 关系背景

- 关系时间线（关键阶段划分）
- 当前状态
- 关系演变趋势

### Layer 2: 对方 6 层人格画像

基于 `persona_analyzer.md` 的 6 层结构输出：
- Layer 0：硬规则（when X then Y）
- Layer 1：身份定位
- Layer 2：表达风格（含原话示例）
- Layer 3：决策模式
- Layer 4：关系行为
- Layer 5：边界红线

### Layer 3: 我的 6 层人格画像

基于 `persona_analyzer.md` 的 6 层结构输出（同上结构）。

### Layer 4: 互动模式

基于 `interaction_analyzer.md` 输出。

### Layer 5: 依恋信号

基于 `attachment_analyzer.md` 输出。

### Layer 6: 沟通建议（场景剧本格式）

参考 `prompts/coach_mode.md` 的场景剧本格式：
- **场景剧本**：每个场景含触发条件、对方心理、推荐做法、禁止事项
- **日常聊天**：话题建议 + 符合对方风格的示例开场
- **暧昧推进**（如适用）：低压力增进方式
- **冲突修复**：具体修复步骤 + 话术
- **消息草稿**：2-3 个版本（稳妥/直接/降压），每个解释为什么这样说

反 AI 腔约束：禁止空泛建议，每条建议必须引用具体聊天记录。

### Layer 7: 不确定性说明

列出所有分析的：
- 低置信度结论
- 缺失的反证
- 替代解释
- 需要更多数据验证的假设

## 主报告 vs 子报告

| 类型 | 触发条件 | 必须包含 | 不包含 |
|------|---------|---------|--------|
| 主报告（bondlens_report_*.md） | 用户要求完整分析 | Layer -1 行动卡 + Layer 0-7 全部 | — |
| 子报告（*_v4.md） | 专项深度分析 | 仅该专项内容 | Layer -1 行动卡、覆盖声明（由主报告承担） |

行动卡（Layer -1）只出现在主报告第一屏。子报告开头引用主报告的行动卡链接，不重复。

## 证据引用规范

**每条核心结论必须严格使用以下格式**（不可省略任何字段）：

```markdown
### {结论标题}

**观察**: {数据中看到了什么——具体行为/消息模式}
**证据**: E-YYYYMMDD-NNN, E-YYYYMMDD-NNN
**推断**: {从观察和证据中推断出什么}
**置信度**: 高|中|低
**反证**: {支持相反结论的证据，或"无强反证"}
**替代解释**: {至少 1 个其他可能的解释}
**建议**: {基于此结论的行动建议，如适用}
```

**注意**：
- 证据 ID 必须存在于 `evidence_index.jsonl` 中
- 置信度为"低"的结论必须在 Layer 7 中再次列出
- 每条结论必须有替代解释，即使你认为概率很低

## 覆盖声明

报告开头必须包含覆盖声明，格式：

```markdown
**数据范围**: {起始日期} 至 {结束日期}
**总消息量**: {N} 条
**抽样说明**: 全量分析 / 抽样 {N} 条（时间窗口：{说明}）
**分析覆盖**: {哪些时间段有充分数据，哪些时间段数据不足}
```

如果分析基于大数据集（>10,000 条消息），必须说明：
1. 是否为全量分析
2. 如果是抽样，抽样方法和时间窗口
3. 哪些时间段的数据覆盖较弱

## 风险词禁用

以下词汇禁止在报告中使用：

| 类型 | 禁用词 | 替代写法 |
|------|--------|---------|
| 过度断言 | 肯定、一定、必然 | "高置信度"、"强烈信号" |
| 伪心理学标签 | 吃醋、冷落测试、PUA | 描述具体行为，不贴标签 |
| 贴标签 | 渣男、渣女、舔狗、备胎 | 描述行为模式 |
| 绝对化 | 总是、从来、永远、每次 | "高频"、"在观察样本中" |

## 语言规范

使用：
> 聊天记录呈现某些……信号。该判断置信度为……。替代解释包括……。

禁止：
> 对方就是……型。
> 对方一定……。
> 这能让对方离不开你。
> 对方肯定……。

```markdown
# BondLens 关系分析报告

**分析对象**: {self_name} ↔ {target_name}
**数据范围**: {date_range}
**总消息量**: {total_messages}
**抽样说明**: 全量分析 / 抽样说明
**分析日期**: {analysis_date}
**框架版本**: BondLens v4.2.0

---

## 先看这个：关系行动卡
{参考 action_brief_builder.md，压缩版行动卡，报告第一屏}

## Layer 0: 核心互动规则
{3-5 条规则，每条严格使用结论格式}

## Layer 1: 关系背景
{时间线 + 当前状态 + 覆盖声明}

## Layer 2: {target_name} 人格画像（6 层结构）
{6 层人格画像 + 原话锚定 + 每条规则使用结论格式}

## Layer 3: {self_name} 人格画像（6 层结构）
{6 层人格画像 + 原话锚定 + 每条规则使用结论格式}

## Layer 4: 互动模式
{正向/负向循环 + 冲突/修复路径 + 每个模式使用结论格式}

## Layer 5: 依恋信号
{双方依恋假设，使用结论格式}

## Layer 6: 沟通建议（场景剧本）
{场景剧本 + 消息草稿，每个场景引用证据 ID}

## Layer 7: 不确定性说明
{低置信度结论 + 替代解释 + 缺失数据}

- B: 证据索引（引用 evidence_index.jsonl 中的所有证据 ID）
```
