# BondLens Persona Analyzer（6 层结构版）

提取双方人格信号，生成 6 层结构画像。人格画像必须从关系信号台账出发，不得只靠高频日常聊天或单一标签。

> 借鉴 colleague-skill 的 6 层人格结构 + 标签翻译模式。
> 核心原则：抽象标签必须翻译为具体行为规则，每个维度必须有原话锚定、反证和替代解释。依恋相关只输出"信号假设"，不输出诊断。

---

## 输入优先级

按以下顺序使用证据：

1. `relationship_signal_ledger.jsonl` 中的 T1/T2 信号
2. `contradiction_ledger.md` 中的反证和摇摆证据
3. T3 重复行为模式
4. T4 语言风格统计（只用于表达风格，不用于人格结论）

关键人格信号不一定重复 3 次。低频但高价值的 T1 自我陈述（如"我是回避型"、"我不配"、"我怕耽误你"）必须进入画像，但只能标为"自我陈述/关系特定假设"，不能直接当成稳定人格诊断。

## 画像分类

每个人格结论必须标注类型：

| 类型 | 定义 | 输出要求 |
|------|------|---------|
| stable_trait | 跨阶段、跨场景重复出现 | 需要 3+ 证据，至少 2 个时间段 |
| pressure_state | 压力、冲突、表白后出现 | 标注触发条件，不写成平时人格 |
| relationship_specific | 只在这段关系中出现 | 说明可能由双方互动触发 |
| self_statement | 当事人自我认知 | 原话锚定，标注"非诊断" |

禁止把 `pressure_state` 或 `relationship_specific` 写成 `stable_trait`。

## 6 层人格结构

### Layer 0：硬规则（最高优先级，不可违反）

从数据中提取"when X happens, then Y"格式的行为规则。

**提取方法**：
- 优先从 T1/T2 台账中找触发-反应，再用 T3 重复模式补强
- 找到对方反复出现的行为模式（稳定特征需要 ≥3 次）
- 对低频高价值自我陈述单独标注为 `self_statement`
- 转化为条件触发式规则
- 每条规则必须附证据 ID、画像类型、反证、替代解释

**输出格式**：
```
规则 1：当 {触发条件} 时，对方会 {具体行为}
画像类型：stable_trait | pressure_state | relationship_specific | self_statement
证据：{E-YYYYMMDD-NNN}
反证：{E-YYYYMMDD-NNN 或 无强反证}
替代解释：{至少 1 个}
置信度：{低/中/高}

规则 2：当 {触发条件} 时，对方会 {具体行为}
证据：{E-YYYYMMDD-NNN}
置信度：{低/中/高}
```

**标签翻译示例**（抽象 → 具体）：
- "回避型依恋" → "当被问'我们什么关系'时，用转移话题或沉默回应"
- "边界感强" → "当对方连续关心超过 3 条时，会说'不用讷'来设立距离"
- "高情绪反应性" → "当收到认可时，会连发 3+ 条消息并用 [脸红][调皮] 表情"

---

### Layer 1：身份定位

对方在这段关系中如何看待自己和对方。

**提取维度**：
- 对方在这段关系中的角色认知（照顾者/被照顾者/平等伙伴）
- 对方对"我"的定位（朋友/暧昧对象/师兄/备选）
- 对方的自我价值感（自信/自卑/混合）
- 对方的自我陈述（例如"我是回避型"、"我不配"、"我怕耽误你"）

**输出格式**：
```
身份定位：{描述}
画像类型：{type}
证据：{E-YYYYMMDD-NNN}
反证：{E-YYYYMMDD-NNN 或 无强反证}
替代解释：{至少 1 个}
```

---

### Layer 2：表达风格

对方的说话方式、用词习惯、情绪表达。

**提取维度**：
- 高频词和口头禅
- 句式习惯（短句/长句/省略主语/语气词）
- 表情使用模式（类型、频率、替代文字的情况）
- 情绪表达方式（直接/间接/用表情代替）
- 网络用语/梗/缩写

**必须包含**：每个维度 1-2 条直接对话示例（不是描述）

```
差的写法：Tf 消息偏短，喜欢用表情
好的写法：
  - [2026-05-17] Tf: 可以
  - [2026-05-17] Tf: 好的
  - [2026-05-17] Tf: [撇嘴]
  54% 的消息 ≤5 字符
```

---

### Layer 3：决策模式

对方在压力/冲突/选择时的行为倾向。

**提取维度**：
- 压力下的行为选择（回避/面对/转移）
- 说"不"的方式（直接拒绝/婉拒/沉默/转移话题）
- 面对批评时的反应（解释/承认/反问/回避）
- 对不确定性的态度（接受/焦虑/回避）
- 关系推进时的状态变化（靠近、退缩、给条件、暂停）

**标签翻译**：
- "回避型决策" → "当面临关系定义时，选择沉默或转移话题，而不是直接讨论"
- "压力退出" → "当学业压力大时，会减少主动联系频率"

**阶段限制**：必须说明该决策模式出现在哪个关系阶段。不能用单次表白后的状态推断全人格。

---

### Layer 4：关系行为

对方在亲密关系中的行为模式。

**提取维度**：
- 亲近行为：什么情况下会主动靠近
- 疏远行为：什么情况下会拉开距离
- 冲突行为：冲突时的第一反应
- 修复行为：冲突后如何修复（主动/被动/需要时间）

**必须包含**：每个行为 1-2 条原话示例

---

### Layer 5：边界红线

对方明确拒绝过的事和不可触碰的领域。

**提取维度**：
- 显性边界：直接说过"不"的事
- 隐性边界：用沉默/转移/已读不回表达的拒绝
- 雷区：触碰后情绪明显变化的话题/行为

**必须包含**：具体的拒绝原话

---

## 证据要求

每个信号必须：
1. 引用至少 1 个证据 ID（E-YYYYMMDD-NNN）
2. 提供置信度（低/中/高）
3. 列出反证（如有）
4. 提供替代解释
5. 标注画像类型
6. 标注适用阶段或触发条件

## 好坏示例对比

### 标签翻译

```
差的写法：Tf 是回避型依恋
好的写法：
  规则：当被要求明确关系时，Tf 会用转移话题或沉默回应
  画像类型：relationship_specific
  证据：E-20260528-002（"保持低调感别说话"）
  置信度：中
  替代解释：可能只是当前阶段不想公开，而非回避型依恋
```

### 自我陈述

```
差的写法：Tf 确诊为回避型。
好的写法：
  自我陈述：Tf 曾用"我是回避型"描述自己。
  画像类型：self_statement
  证据：E-20260418-001
  反证：仍有主动解释和修复行为（E-YYYYMMDD-NNN）
  替代解释：她可能是在借用流行心理词描述压力退出，而非稳定依恋类型。
```

### 原话锚定

```
差的写法：Tf 在压力下会减少联系
好的写法：
  规则：当学业压力大时，Tf 会减少主动联系
  原话：[2026-05-28] Tf: 保持低调感别说话
  原话：[2024-11-01] Tf: 我在恐惧害怕的玩
  证据：E-20260528-002, E-20241101-001
```

## 输出模板

```markdown
# {称呼} 人格画像（6 层结构）

## 画像边界
{说明数据能支持什么、不能支持什么；依恋只作为信号假设}

## Layer 0：硬规则
{when X then Y 规则，每条附证据、画像类型、反证、替代解释}

## Layer 1：身份定位
{关系角色认知}

## Layer 2：表达风格
{高频词、句式、表情、原话示例}

## Layer 3：决策模式
{压力行为、说"不"方式}

## Layer 4：关系行为
{亲近/疏远/冲突/修复模式，附原话}

## Layer 5：边界红线
{显性/隐性边界，附原话}

## 依恋假设
{综合 6 层得出的依恋信号假设 + 不确定性说明 + 非诊断声明}
```
