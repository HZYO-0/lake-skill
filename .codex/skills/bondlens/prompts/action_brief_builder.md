# BondLens Action Brief Builder

生成"关系行动卡"——报告的第一屏，让用户 1 分钟内知道"我现在该怎么做"。

> 核心原则：行动卡是给"不想读长报告"的人看的。每条建议必须有触发条件、具体动作、证据 ID、禁忌边界。

---

## 策略路由器

根据数据自动选择当前策略，不固定为单一风格：

| 策略 | 适用条件 | 选择信号 |
|------|---------|---------|
| **低压稳定** | 关系不明、对方变短变慢、情感话题触发回避、关系暴露敏感 | Tf 回复 >2 小时、消息 ≤3 字符、情感话题后结束对话 |
| **适度推进** | 对方近期有主动靠近、持续回应、接受轻度暧昧 | Tf 主动发起话题、回复及时且内容丰富、接受 Zy 的关心不回避 |
| **冲突修复** | 数据出现误会、冷淡、压力升级、边界被触碰 | 明显冷淡期、Tf 表达不满、Zy 踩到雷区 |

**默认策略**：如果证据冲突或不确定，降级为 `低压稳定`，并在行动卡里写明不确定性。

**策略选择必须引用证据**：说明为什么选这个策略，基于哪些数据信号。

## 每条建议的质量标准

每条行动建议必须满足以下 4 个条件，缺一不可：

| 条件 | 说明 | 检查方式 |
|------|------|---------|
| 触发条件 | 什么情况下用这个建议 | 必须写明"当……时" |
| 具体动作 | 做什么、说什么、不做什么 | 不能是"保持适度联系"这种泛话 |
| 证据 ID | 引用 evidence_index.jsonl 中的 ID | 格式 E-YYYYMMDD-NNN |
| 禁忌边界 | 用这个建议时不要做什么 | 必须写明"不要……" |

### 好坏对比

```
差的写法：
建议保持适度联系，给她一些空间。

好的写法：
这两天别主动找她。她答辩完 + 上次表白的事还在消化，现在需要自己回血。
第三天发个食堂吐槽："今天那个红烧肉，我怀疑是用橡皮做的"——用你们平时的吐槽方式破冰。
不要问"你怎么不理我"或"是不是我上次说错话了"。
证据：E-20260601-002, E-20260604-002
```

## 话术生成规则

生成可直接发送的消息时，必须遵守：

1. **风格匹配**：消息必须符合对方的语言风格——用对方会用的词、句式、表情
2. **引用锚点**：从历史聊天中找到类似场景的消息，作为风格参考
3. **三个版本**：
   - 稳妥版：最低风险，不会出错
   - 降压版：如果对方最近压力大或回复变慢
   - 更直接版：如果关系进展顺利，可以稍微推进
4. **每个版本解释**：为什么这样说、适合什么场景

### 禁止的话术

- 任何像"AI 心理咨询师"的话
- 任何空泛的、可以套用到任何关系的话
- 任何带有操控意图的话（试探、施压、诱导）
- 任何对方不使用的词汇或句式

---

## 与长手册的关系

- **行动卡**：默认输出，压缩版，前置在报告第一屏
- **长手册**（communication_playbook_v4.md 格式）：只在用户明确要"完整手册""剧本版""详细建议"时生成
- 行动卡的内容从长手册中提炼，但格式更紧凑、更可执行

---

## 安全边界

- 不预测关系结局
- 不诊断人格类型
- 不使用确定性语言（"肯定""一定"）
- 不建议操控策略（PUA、冷落测试、嫉妒诱导）
- 不在对方明确拒绝后建议继续追求
- 不建议冷暴力、消失测试
- 策略路由必须基于证据，不能凭空选择
