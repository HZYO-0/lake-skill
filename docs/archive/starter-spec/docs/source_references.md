# Source References for Project Documentation

Use these as background references for README/source-card wording. Do not dump long copyrighted text into the Skill.

## OpenAI / ChatGPT privacy and data controls

- OpenAI Privacy Policy: https://openai.com/policies/privacy-policy/
- OpenAI Data Controls FAQ: https://help.openai.com/en/articles/7730893-data-controls-faq

Project implication:

- User uploads, prompts, files, images, audio/video, and other content may be processed as user content depending on product settings.
- Hybrid mode should minimize uploads by sending redacted digest and evidence index, not raw chat databases.

## GitHub Actions

- Writing workflows: https://docs.github.com/en/actions/how-tos/write-workflows
- Secure use reference: https://docs.github.com/en/actions/reference/security/secure-use

Project implication:

- CI should run lint, type checks, tests, privacy scans, and package validation.
- Workflows should use least permissions and security checks.

## Personality / attachment / symbolic methods

- Big Five personality traits overview: https://en.wikipedia.org/wiki/Big_Five_personality_traits
- Attachment measures overview: https://en.wikipedia.org/wiki/Attachment_measures
- Barnum effect overview: https://en.wikipedia.org/wiki/Barnum_effect
- Carlson, S. A double-blind test of astrology. Nature 318, 419-425 (1985): https://www.nature.com/articles/318419a0

Project implication:

- Use Big Five and attachment only as non-clinical communication-signal frameworks.
- Do not use astrology/tarot for evidence-based conclusions.
- Guard against vague statements that feel personally accurate but are unsupported by evidence.
