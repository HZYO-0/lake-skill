# LakeSkill 湖镜 0.12 宣传方案

> 公开传播只使用合成数据；先让用户运行 Demo，再决定是否导入自己的聊天记录。

## 主定位

别让 AI 猜 TA 爱不爱你。让 LakeSkill 告诉你：聊天证据支持什么、不支持什么、下一步怎么做更稳。

## 冷启动顺序

1. **GitHub/Skill 安装转化**：先把 README 和 INSTALL 做到 5 分钟内能装好、能跑通 Demo。
2. **小红书破圈**：图文/长图为主，用"聊天焦虑纠偏"切入情感内容入口。
3. **抖音短视频放大**：20-45 秒录屏，用"AI 说得太肯定"的痛点钩子引流。

商业化只做：隐私优先的数据整理、安装部署、报告解读模板。不做挽回咨询/情感陪伴 SaaS。

## 抖音内容矩阵

### 视频模板（35 秒）

```
[0-3s]  钩子：痛点大字 + 反转
[3-8s]  引出工具：一句话定位
[8-15s] 展示输入：合成聊天 CSV
[15-25s] 展示核心输出：行动卡 + 证据 ID
[25-35s] CTA：GitHub 搜索 + 安装口令
```

### 首批标题（8 条）

1. 把聊天记录丢给 AI，最怕它说得太肯定
2. 我做了一个不会替你上头的关系分析工具
3. 聊天记录太多？先做数据体检再分析
4. TA 回得慢 = 不在乎？AI 不应该帮你下这个结论
5. 每条关系判断都要有证据 ID，没有就是猜
6. 证据不够时，AI 应该降级，而不是把话说满
7. 聊天记录给 AI 前，先做这件事
8. 为什么我不做挽回神器

### 不投放

- 不说"帮你复合"
- 不说"判断真爱"
- 不说"识别回避型人格"
- 不说"让 TA 离不开你"

## 合规边界

- 不扮演对方，不模拟持续陪伴
- 不制造情感依赖
- 不提供操控策略
- 如涉及自伤/伤害他人/被威胁/被跟踪，优先提示现实支持系统
- 不面向未成年人亲密陪伴服务
- 如果未来面向中国境内公众提供持续性情感互动服务，按当时有效法规重新做合规评估
