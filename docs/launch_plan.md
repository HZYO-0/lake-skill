# LakeSkill 0.12 发布与传播执行方案

## 发布原则

产品闭环先于流量。全部公开案例只使用合成数据；不宣传爱情判断准确率；隐私文案明确区分本地预处理、脱敏后上传和本地模型全程处理。

统一 CTA：

> GitHub 搜索 LakeSkill 湖镜，先运行合成 Demo，再决定是否导入自己的聊天记录。

## 首批六篇

1. AI 最容易漏掉的，不是回复速度，而是你们真正谈过什么。
2. 不是不喜欢你，只是不想现在开始：为什么不能压成接受或拒绝？
3. 明确拒绝后仍然每天聊天，AI 为什么不能说其实喜欢？
4. 我拆了一个热门恋爱分析项目：哪些值得学，哪些不能学？
5. 从十万条聊天里找全关系定义，而不是让模型随便抽样。
6. 聊天记录交给 AI 前，怎样先在本地脱敏？

## 节奏

发布前 7 天：完成 README、四套合成报告、4 张视觉稿和录屏；发布两篇问题教育与一篇构建过程；仅用合成或脱敏数据测试。

发布日：GitHub Release 与 PyPI 同步；小红书、抖音、B站同步；主帖展示关系讨论置顶前后对比。

发布后 30 天：每周两个合成案例和一个技术/隐私内容；整理失败案例；发布“哪些情况下 LakeSkill 会明确说不知道”；默认不加入产品遥测。

## 指标

产品：5 分钟 Demo、process 成功率、报告成功率、决策完整率 100%、T1 覆盖率 100%、编码/打包/漂移通过率 100%。

传播：README 到安装行为、star/fork/issue、PyPI 趋势、小红书收藏率、视频完播率、Demo 反馈与高频失败点。
