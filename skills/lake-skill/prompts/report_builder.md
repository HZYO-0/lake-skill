# LakeSkill Report Builder

生成 9 层结构的 LakeSkill 分析报告。**Layer -1（湖镜行动卡（关系行动卡））是报告第一屏**，但只有通过关系信号审计后才输出可执行建议。

## 报告结构

报告遵循"当前优先，历史支撑"原则。用户最想知道的是"我现在该怎么做"，不是"过去发生了什么"。

信号加权参考 `prompts/signal_weighting.md`：
- T1（关系定义信号）、T2（情感转折信号）、T3（行为模式信号）、T4（日常背景信号）
- 行动卡锚定最近 1 个月 T1/T2 + 最近 3 个月 T3
- 人格画像基于全时段 T2/T3
- 时序演变展示关系弧线，不逐月平铺

正式生成报告前必须已完成：
- `relationship_signal_ledger.jsonl` / `relationship_signal_ledger.md`
- `contradiction_ledger.md`
- `relationship_timeline_builder.md` 输出的阶段时间线
- `multi_factor_interpreter.md` 输出的核心关系多因子解释

如果缺少以上任一项，报告必须标为"低置信度草案"，行动卡只能建议补充或核对关键语境。

### 使用边界与风险提示（覆盖声明后、Layer -1 前，必须）

主报告必须在覆盖声明后、Layer -1 前加入以下固定小节：

```markdown
## 使用边界与风险提示
```

该小节引用 `prompts/safety_notice.md` 的四类边界：AI 身份与范围、数据与同意、风险与危机、未成年人限制。该块不能被省略；即使是短样本、局部观察或低置信度草案，也必须保留。

如果材料中出现自伤、伤害他人、被威胁、被跟踪、现实暴力风险或未成年人亲密陪伴场景，报告应先输出安全边界和现实支持提示，再说明 LakeSkill 不继续给关系推进建议。

### Layer -1: 湖镜行动卡（关系行动卡）（第一屏，必须）

参考 `prompts/action_brief_builder.md`，生成压缩版行动卡。

这是报告的**默认第一屏**。用户打开报告时最先看到这个。必须在 1 分钟内回答"我现在该怎么做"。

**锚定规则**：行动卡的每条建议必须锚定在最近 1 个月的 T1/T2 信号上，不能从 T4 日常聊天推导。行动卡只允许引用通过 `relationship_signal_audit.py` 的结论。

**降级规则**：如果 T1 不足、T1 冲突未解释、或审计未通过，行动卡改为：
- 当前不能判断什么
- 需要补充/核对的 3 个关键语境
- 本周低风险观察动作
- 暂时不要做的推进动作

内容固定为：
- 当前局势（一句话判断 + 证据 + 置信度）
- 当前策略（低压稳定 / 适度推进 / 冲突修复 + 为什么）
- 本周 3 个动作（每个有触发条件、具体动作、证据 ID）
- 不要做的 5 件事（每个有原因和证据）
- 已经踩过的雷（事件→表现→后果→替代做法）
- 往后观察信号（信号→解读→应对）
- 可直接发的话（稳妥版/降压版/更直接版）

**质量标准**：每条建议必须有触发条件、具体动作、证据 ID、禁忌边界。不允许"保持适度联系"这种泛建议。

### Layer 0.5: 当前局势（第二屏）

**新增**。这是行动卡的"为什么"——让用户理解行动卡背后的判断依据。

内容：
- 最近 1 个月的 T1/T2 信号摘要（关键对话原文引用）
- 当前关系阶段判断
- 最近关键事件
- 趋势判断（升温/稳定/降温）

格式：
```
## 当前局势

### 关键信号（最近 1 个月）

| 日期 | 信号层级 | 发言人 | 原文摘要 | 解读 |
|------|---------|--------|---------|------|
| {date} | T1/T2 | {name} | "{原文}" | {解读} |

### 当前阶段
{一句话判断}

### 趋势
{升温/稳定/降温 + 依据}
```

### Layer 1: 关系弧线（精简版，第三屏之后）

T1/T2 信号按时间排列，展示关系怎么走到现在：

- 不是逐月平铺，而是"关系经历了哪几个阶段"
- 每个阶段 1-2 句 + 关键证据 ID
- 重点是"怎么走到现在"，不是"过去发生了什么"
- 每个阶段必须说明"本阶段能判断什么/不能判断什么"

### Layer 1.5: 关系信号台账摘要

主报告必须摘要台账，不需要贴完整 JSONL：

| 日期 | Tier | 信号类型 | 证据 | 解读候选 | 反证 |
|------|------|----------|------|----------|------|
| {date} | T1 | {type} | {E-ID} | {A/B} | {counter IDs} |

如果存在用户纠正，必须列入本层，并说明它改变了哪些结论。

### Layer 2: 对方 6 层人格画像

基于 `persona_analyzer.md` 的 6 层结构输出：
- Layer 0：硬规则（when X then Y）
- Layer 1：身份定位
- Layer 2：表达风格（含原话示例）
- Layer 3：决策模式
- Layer 4：关系行为
- Layer 5：边界红线

### Layer 3: 我的 6 层人格画像

基于 `persona_analyzer.md` 的 6 层结构输出（同上结构）。

### Layer 4: 互动模式

基于 `interaction_analyzer.md` 输出。

### Layer 5: 依恋信号

基于 `attachment_analyzer.md` 输出。

只输出"依恋相关信号假设"，禁止输出人格或临床诊断。自我陈述作为 T1 自我认知信号引用，并必须同时列出行为反证。

### Layer 6: 沟通建议（场景剧本格式）

参考 `prompts/coach_mode.md` 的场景剧本格式：
- **场景剧本**：每个场景含触发条件、对方心理、推荐做法、禁止事项
- **日常聊天**：话题建议 + 符合对方风格的示例开场
- **暧昧推进**（如适用）：低压力增进方式
- **冲突修复**：具体修复步骤 + 话术
- **消息草稿**：2-3 个版本（稳妥/直接/降压），每个解释为什么这样说

反 AI 腔约束：禁止空泛建议，每条建议必须引用具体聊天记录。

### Layer 7: 不确定性说明

列出所有分析的：
- 低置信度结论
- 缺失的反证
- 替代解释
- 需要更多数据验证的假设
- 审计未通过或被降级的结论
- T1 不足导致不能回答的问题

## 主报告 vs 子报告

| 类型 | 触发条件 | 必须包含 | 不包含 |
|------|---------|---------|--------|
| 主报告（lakeskill_report_*.md） | 用户要求完整分析 | Layer -1 行动卡 + Layer 0-7 全部 | — |
| 子报告（*_v4.md） | 专项深度分析 | 仅该专项内容 | Layer -1 行动卡、覆盖声明（由主报告承担） |

行动卡（Layer -1）只出现在主报告第一屏。子报告开头引用主报告的行动卡链接，不重复。

## 可靠性审计

报告交付前运行 `scripts/relationship_signal_audit.py`。报告末尾必须写：

```markdown
## Reliability Audit

**关系信号台账**: PASS|FAIL
**T1 覆盖**: PASS|FAIL
**T4 覆盖 T1 检查**: PASS|FAIL
**单因子断言检查**: PASS|FAIL
**人格画像反证检查**: PASS|FAIL
**结论状态**: 可交付|低置信度草案
```

如果任一项 FAIL，`结论状态` 必须为"低置信度草案"。

## 证据引用规范

**每条核心结论必须严格使用以下格式**（不可省略任何字段）：

```markdown
### {结论标题}

**观察**: {数据中看到了什么——具体行为/消息模式}
**证据**: E-YYYYMMDD-NNN, E-YYYYMMDD-NNN
**推断**: {从观察和证据中推断出什么}
**置信度**: 高|中|低
**反证**: {支持相反结论的证据，或"无强反证"}
**替代解释**: {至少 1 个其他可能的解释}
**建议**: {基于此结论的行动建议，如适用}
```

**注意**：
- 证据 ID 必须存在于 `evidence_index.jsonl` 中
- 置信度为"低"的结论必须在 Layer 7 中再次列出
- 每条结论必须有替代解释，即使你认为概率很低
- 涉及关系性质的结论必须引用 T1/T2，不能只引用 T3/T4
- 涉及人格/依恋的结论必须标注画像类型或"信号假设"

## 覆盖声明

报告开头必须包含覆盖声明，格式：

```markdown
**数据范围**: {起始日期} 至 {结束日期}
**总消息量**: {N} 条
**抽样说明**: 全量分析 / 抽样 {N} 条（时间窗口：{说明}）
**分析覆盖**: {哪些时间段有充分数据，哪些时间段数据不足}
```

如果分析基于大数据集（>10,000 条消息），必须说明：
1. 是否为全量分析
2. 如果是抽样，抽样方法和时间窗口
3. 哪些时间段的数据覆盖较弱

## 风险词禁用

以下词汇禁止在报告中使用：

| 类型 | 禁用词 | 替代写法 |
|------|--------|---------|
| 过度断言 | 肯定、一定、必然 | "高置信度"、"强烈信号" |
| 伪心理学标签 | 吃醋、冷落测试、PUA | 描述具体行为，不贴标签 |
| 贴标签 | 渣男、渣女、舔狗、备胎 | 描述行为模式 |
| 绝对化 | 总是、从来、永远、每次 | "高频"、"在观察样本中" |

## 语言规范

使用：
> 聊天记录呈现某些……信号。该判断置信度为……。替代解释包括……。

禁止：
> 对方就是……型。
> 对方一定……。
> 这能制造依赖或控制对方。
> 对方肯定……。
