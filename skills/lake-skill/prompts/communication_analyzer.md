# LakeSkill Communication Pattern Analyzer

分析双方沟通模式，产出沟通模式摘要。

## 分析维度

### 1. 响应模式

提取指标：
- **响应时间分布**：中位数、P25、P75（排除睡眠时间）
- **消息长度分布**：字符数中位数、≤5 字符占比
- **话题开启率**：谁更常发起新话题
- **连续消息率**：谁更常连发多条

输出格式：
```
[{target_name}] 响应时间中位数: X 分钟（P25: X, P75: X）
[{target_name}] 消息长度中位数: X 字符，≤5 字符占比: X%
[{self_name}] 响应时间中位数: X 分钟（P25: X, P75: X）
[{self_name}] 消息长度中位数: X 字符，≤5 字符占比: X%
话题开启: {target_name} X% / {self_name} X%
```

### 2. 情绪表达模式

识别信号词和模式：
- **高情绪词汇**：哈哈哈、啊啊啊、呜呜呜、！！！、？？？
- **表情符号使用频率和类型**
- **感叹号/问号密度**
- **大写/重复字符使用**

输出：双方情绪表达风格对比

### 3. 话题模式

分类话题类型：
- 日常分享（吃饭、天气、出行）
- 工作/学业（论文、项目、导师）
- 情感表达（关心、思念、抱怨）
- 信息交换（链接、文件、教程）
- 调侃/玩笑

输出：话题偏好雷达图（文字版）

### 4. 边界模式

识别边界信号：
- **显性边界**："不要"、"不想"、"别"
- **隐性边界**：已读不回、话题转移、简短回复
- **边界触发场景**：哪些话题/行为触发边界
- **边界后的恢复时间**

输出：边界信号表

### 5. 非字面表达模式（"搞抽象"识别）

识别非字面、反讽、调侃等需要上下文理解的表达：

| 模式 | 识别方式 | 示例 |
|------|---------|------|
| **反讽/自嘲** | 字面负面但语境轻松，通常伴随"哈哈哈"或表情 | "死啊" + 哈哈哈哈；"完蛋了" + [吃瓜] |
| **表情代替文字** | 用 [表情] 代替直接表达情绪，不附文字 | [委屈] [撇嘴] [脸红] 单独发送 |
| **用梗/网络语** | 缩写、谐音、二次元用语、平台梗 | "bushi""xswl""nmd""6" |
| **故意夸张** | 明显不符合事实的夸张表达 | "我要死了""完蛋了""啊啊啊啊啊" |
| **转移式回应** | 用玩笑化解严肃话题，或用调侃代替直接回答 | 岔开话题 + 调侃；用表情包结束严肃讨论 |
| **省略式表达** | 用"......""。。。。"或单个标点表达复杂情绪 | "。。。"可能表示无语/无奈/不想继续 |
| **反向表达** | 说"不要"但语境是撒娇，说"讨厌"但语气是亲密 | "不许骑车"（实际是关心） |

**提取方法**：
1. 找到字面意思和语境不一致的消息
2. 结合前后文判断真实意图
3. 记录为"字面 → 实际含义"对
4. 附原话作为证据

**输出格式**：
```
原话：[日期] {发送者}: "{字面表达}"
字面含义：{字面理解}
实际含义：{结合语境的理解}
语境：{前后文}
证据：{E-YYYYMMDD-NNN}
```
