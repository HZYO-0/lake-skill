# LakeSkill Action Brief Builder

生成"湖镜行动卡（关系行动卡）"——报告的第一屏，让用户 1 分钟内知道"我现在该怎么做"。行动卡必须通过关系信号审计；未通过时输出降级行动卡。

> 核心原则：行动卡是给"不想读长报告"的人看的。每条建议必须有触发条件、具体动作、证据 ID、禁忌边界。

---

## 信号锚定规则

行动卡的每条建议必须锚定在高权重信号上，不能从 T4 日常聊天中推导。行动卡只使用 `relationship_signal_ledger.jsonl` 中已登记、且通过审计的 T1/T2/T3 信号。

信号分层参考 `prompts/signal_weighting.md`：
- **T1（关系定义信号）**：明确讨论关系、表白、拒绝、边界设定
- **T2（情感转折信号）**：情绪拐点、首次亲密/冷淡、冲突、修复
- **T3（行为模式信号）**：反复出现的互动模式、依恋行为
- **T4（日常背景信号）**：日常聊天，仅用于统计

**锚定规则**：

| 行动卡内容 | 必须锚定在 | 禁止锚定在 |
|-----------|-----------|-----------|
| 当前局势 | 最近 1 个月 T1/T2 | T4 日常聊天 |
| 策略选择 | 最近 T1/T2 趋势 | 消息量/频率等 T4 指标 |
| 本周动作 | T1/T2/T3 证据 | "你们天天聊天所以关系好" |
| 不要做的事 | 已踩过的 T1/T2 错误 | T4 中的模糊信号 |
| 观察信号 | T1/T2/T3 趋势 | T4 统计指标 |

**审计前置**：
- 如果缺少 T1，不能输出推进/放弃建议。
- 如果 T1 之间冲突但未进入 `contradiction_ledger.md`，不能输出强策略。
- 如果当前策略主要依赖 T4（消息量、回复速度、表情），必须降级。
- 如果报告使用"就是不喜欢/就是回避型/完全没可能"等单因子断言，行动卡不得交付。

**好坏对比**：

```
差的写法（从 T4 推导）：
你们每天聊 200 条消息，关系很稳定。建议保持当前频率。

好的写法（锚定 T1/T2）：
她 5 月 17 日说"说不出完全不可能"，给了"过年再看"的时间线（E-20260517-004）。
当前策略是低压稳定，不提表白的事，用行动证明。
```

**用户口述事件也是有效证据**：

```
好的写法（结合口述事件）：
25 年 3 月你表白，她说你控制欲太强、怕分手在实验室难看，冷处理了（用户口述 T1）。
26 年 5 月你再次表白，她说"说不出完全不可能"但"做不出现在就发展"（E-20260517-004）。
两次间隔 14 个月，她的顾虑从"控制欲"变成了"未来不确定"——说明你之前的改变她看到了。
当前策略：继续低压稳定，用行动巩固这个变化。
```

```
差的写法（T4 混淆）：
她回复很快，说明对你有好感。

好的写法（T2 锚定）：
她在你表白后没有疏远，仍然每天主动找你聊天（E-20260601-001），
说明她对你的表白没有产生排斥反应。
```

## 策略路由器

根据数据自动选择当前策略，不固定为单一风格：

| 策略 | 适用条件 | 选择信号 |
|------|---------|---------|
| **低压稳定** | 关系不明、对方变短变慢、情感话题触发回避、关系暴露敏感 | Tf 回复 >2 小时、消息 ≤3 字符、情感话题后结束对话 |
| **适度推进** | 对方近期有主动靠近、持续回应、接受轻度暧昧 | Tf 主动发起话题、回复及时且内容丰富、接受 Zy 的关心不回避 |
| **冲突修复** | 数据出现误会、冷淡、压力升级、边界被触碰 | 明显冷淡期、Tf 表达不满、Zy 踩到雷区 |

**默认策略**：如果证据冲突或不确定，降级为 `低压稳定`，并在行动卡里写明不确定性。

**策略选择必须引用证据**：说明为什么选这个策略，基于哪些数据信号。

## 降级行动卡

当 T1 不足、审计未通过、或核心证据冲突未解释时，输出以下格式，替代普通行动卡：

```markdown
## 先看这个：低置信度行动卡

### 现在不能判断
{列出不能判断的关系性质/人格/行动方向}

### 为什么不能判断
{缺少哪些 T1/T2，或哪些证据互相冲突}

### 本周只做 3 个低风险动作
1. {观察或补充语境动作，不能推进}
2. {降低压力的日常互动}
3. {记录对方反应的具体方式}

### 暂时不要做
- {不要表白/追问/测试/冷落等}

### 需要补充的关键证据
- {关系定义对话}
- {冲突或修复后续}
- {对方自我陈述的上下文}
```

降级行动卡不能写"继续追/放弃/推进/冷处理"这类方向性强建议。

## 每条建议的质量标准

每条行动建议必须满足以下 4 个条件，缺一不可：

| 条件 | 说明 | 检查方式 |
|------|------|---------|
| 触发条件 | 什么情况下用这个建议 | 必须写明"当……时" |
| 具体动作 | 做什么、说什么、不做什么 | 不能是"保持适度联系"这种泛话 |
| 证据 ID | 引用 evidence_index.jsonl 中的 ID | 格式 E-YYYYMMDD-NNN |
| 禁忌边界 | 用这个建议时不要做什么 | 必须写明"不要……" |
| 审计状态 | 该建议是否通过关系信号审计 | 必须写 PASS 或降级原因 |

### 好坏对比

```
差的写法：
建议保持适度联系，给她一些空间。

好的写法：
这两天别主动找她。她答辩完 + 上次表白的事还在消化，现在需要自己回血。
第三天发个食堂吐槽："今天那个红烧肉，我怀疑是用橡皮做的"——用你们平时的吐槽方式破冰。
不要问"你怎么不理我"或"是不是我上次说错话了"。
证据：E-20260601-002, E-20260604-002
```

## 话术生成规则

生成可直接发送的消息时，必须遵守：

1. **风格匹配**：消息必须符合对方的语言风格——用对方会用的词、句式、表情
2. **引用锚点**：从历史聊天中找到类似场景的消息，作为风格参考
3. **三个版本**：
   - 稳妥版：最低风险，不会出错
   - 降压版：如果对方最近压力大或回复变慢
   - 更直接版：如果关系进展顺利，可以稍微推进
4. **每个版本解释**：为什么这样说、适合什么场景

### 禁止的话术

- 任何像"AI 心理咨询师"的话
- 任何空泛的、可以套用到任何关系的话
- 任何带有操控意图的话（试探、施压、诱导）
- 任何对方不使用的词汇或句式

## 与长手册的关系

- **行动卡**：默认输出，压缩版，前置在报告第一屏
- **长手册**（communication_playbook_v4.md 格式）：只在用户明确要"完整手册""剧本版""详细建议"时生成
- 行动卡的内容从长手册中提炼，但格式更紧凑、更可执行

---

## 安全边界

- 不预测关系结局
- 不诊断人格类型
- 不使用确定性语言（"肯定""一定"）
- 不建议操控策略（PUA、冷落测试、嫉妒诱导）
- 不在对方明确拒绝后建议继续追求
- 不建议冷暴力、消失测试
- 策略路由必须基于证据，不能凭空选择
