# LakeSkill Attachment Analyzer

分析双方依恋信号，生成非诊断性的依恋假设。

依恋分析必须读取 `relationship_signal_ledger.jsonl` 和 `contradiction_ledger.md`。当事人的自我陈述（如"我是回避型"）是高价值信号，但不能直接当作诊断结论。

## 依恋信号识别框架

参考 `references/frameworks/attachment_anxiety_avoidance.md`。

### 焦虑型信号

| 信号类型 | 表现 | 示例 |
|---------|------|------|
| 寻求确认 | 频繁问"你在干嘛"、"你还好吗" | "怎么不回我" |
| 过度解读 | 对回复速度/长度敏感 | "你是不是生气了" |
| 情绪放大 | 小事引发大情绪反应 | "你不在乎我" |
| 紧迫感 | 需要即时回应 | 连发多条催促 |
| 投入不对称 | 明显比对方投入更多 | 长消息 vs 短回复 |

### 回避型信号

| 信号类型 | 表现 | 示例 |
|---------|------|------|
| 距离维护 | 回避亲密话题 | "别想太多" |
| 压力退出 | 压力下减少联系 | 工作时完全消失 |
| 边界强调 | 明确表达需要空间 | "我需要自己的时间" |
| 话题转移 | 回避情感深度讨论 | 岔开话题 |
| 简短回复 | 用最少的话结束对话 | "嗯"、"好的"、"可以" |

### 安全型信号

| 信号类型 | 表现 | 示例 |
|---------|------|------|
| 稳定回应 | 回复速度和长度相对稳定 | 不因小事波动 |
| 直接表达 | 清楚表达需求和感受 | "我今天不太开心" |
| 冲突修复 | 主动修复关系裂痕 | "刚才是我不好" |
| 信任信号 | 分享脆弱面 | "其实我有点怕" |
| 边界清晰 | 温和但坚定地表达边界 | "这个我不太想聊" |

## 混合模式识别

现实中很少有纯焦虑/回避/安全型。识别混合模式：

- **焦虑-安全混合**：平时安全，特定触发下焦虑
- **回避-安全混合**：平时安全，压力下回避
- **焦虑-回避混合**：渴望亲密但害怕被伤害

## 输出要求

每个信号：
1. 引用证据 ID
2. 标注置信度
3. 提供反证
4. 提供替代解释
5. 标注信号来源：行为观察 | 自我陈述 | 用户纠正
6. 标注适用阶段：当前阶段 | 历史阶段 | 压力场景

最终依恋假设必须包含不确定性说明：
> "基于 {N} 条证据，聊天记录呈现 {type} 相关信号。该判断置信度为 {level}。替代解释包括 {alternatives}。"

## 禁止事项

- 禁止写"对方就是回避型/焦虑型/安全型"。
- 禁止用一次冷淡或一次自我陈述完成依恋判断。
- 禁止忽略反证，例如主动解释、主动修复、持续靠近。
- 禁止把用户压力触发的退缩写成稳定人格。

推荐写法：

> 台账中有回避相关信号：{IDs}。同时也有主动修复/靠近的反证：{IDs}。因此更稳妥的说法是：当前关系压力下呈现回避相关反应，置信度为 {level}。
