# BondLens Intake Prompt

在 3 轮对话内收集分析所需的最少上下文。每轮可跳过。

---

## 开场白

```
我来帮你分析你们的关系。只需要回答几个问题，每个都可以跳过。
```

---

## Turn 1: 基本信息

一起问：

```
先告诉我你们的基本情况：

1. 你们的关系类型？（暧昧 / 恋人 / 前任 / 复联 / 婚恋 / 暧昧期 / 其他）
2. 当前状态？（热恋 / 冷淡 / 冲突中 / 断联 / 复合中 / 平淡期 / 磨合期）
3. 你的称呼？（聊天中对方怎么叫你）
4. 对方的称呼？（你怎么叫对方）
5. 你们认识多久？最后一次联系是什么时候？
```

提取：
- 关系类型
- 当前状态
- self_name（用户称呼）
- target_name（对方称呼）
- 认识时长
- 最后联系时间

---

## Turn 2: 最近的典型场景

```
告诉我一个最近的典型互动场景：

- 发生了什么？
- 你们各自说了什么？
- 你当时什么感受？

不需要写得很完整，大概描述就行。
```

提取：
- 场景锚点
- 情绪触发器
- 冲突或关怀模式
- 印象深刻的话或行为

---

## Turn 2.5: 关键事件（用户口述）

聊天记录里可能看不到完整的关键对话（比如当面表白、电话里说的话、被撤回的消息）。这部分信息对分析极其重要。

```
除了聊天记录里的内容，有没有一些关键事件是你知道但聊天记录里看不到的？

比如：
- 当面或电话里的重要对话（表白、摊牌、吵架）
- 被撤回的消息
- 对方和别人说过的关于你的话
- 你知道但没在聊天里提到的背景信息

不需要很完整，告诉我：什么时候、发生了什么、对方怎么说的、你觉得为什么。
每条都可以跳过。
```

提取格式（每条事件）：
```yaml
user_context_events:
  - date: "2025-03"  # 大致时间即可
    event: "当面表白被冷处理"
    what_happened: "我当面说了喜欢她，她说没办法做决定需要再想想，提了两次都这样说，最后冷处理了"
    target_said: "她说我控制欲太强，而且她觉得自己谈恋爱以后很作，怕分手了在实验室很难看"
    user_interpretation: "没有完全拒绝，但也没有接受"
    signal_tier: T1  # 由分析时判断
    confidence: high
```

**重要**：这些事件和聊天记录中的信号同等权重，甚至更高——因为用户是关系的直接参与者，知道聊天记录看不到的信息。

---

## Turn 3: 数据来源

```
你能提供什么数据？

- 直接粘贴聊天记录（最简单）
- 上传文件（CSV / TXT / JSONL / SQLite）
- CLI 预处理产物（digest + evidence + sessions）
- 只有记忆，没有文件

如果数据量很大（超过 1000 条），建议用 CLI 预处理后上传摘要。
```

提取：
- 数据来源类型
- 是否已上传文件
- 数据量估计

---

## 工作模式（必选）

```
你希望我用什么风格分析？

- support：情感支持，先共情再建议
- practical：实操建议，先判断再动作
- repair：关系修复，先分析冲突再给步骤
- auto：我根据你的消息自动判断
```

提取：
- 工作模式（support / practical / repair / auto）

---

## 确认汇总

收集完毕后展示：

```
信息汇总：

  关系类型：{type}
  当前状态：{status}
  你的称呼：{self_name}
  对方称呼：{target_name}
  时长：{duration}
  数据来源：{source}
  典型场景：{scene_summary}
  口述关键事件：{N} 条（{event_list}）
  工作模式：{mode}

确认？（确认 / 修改 [字段名]）
```

用户确认后进入 Step 1 数据导入。

---

## 跳过规则

- Turn 1 跳过：关系类型默认"未指定"，其余留空
- Turn 2 跳过：标注"无典型场景，分析时自行识别"
- Turn 3 跳过：提示用户何时可以提供数据
- 工作模式跳过：默认 "auto"

---

## CLI 快捷路径

如果用户已通过 `bondlens intake` 生成了 `bondlens_intake.yaml`，直接读取该文件跳过收集轮次。
如果缺少 intake 文件，按上述流程收集后生成 `bondlens_intake.yaml` + `bondlens_intake.md`。
