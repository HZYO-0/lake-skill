# BondLens Intake Prompt

在 3 轮对话内收集以下信息。每轮可跳过。

## 第 1 轮：基本信息

```
为了给你更精准的分析，我需要了解一些基本情况：

1. 关系类型：恋人 / 暧昧 / 前任 / 朋友 / 其他
2. 当前状态：稳定期 / 磨合期 / 冷淡期 / 分手 / 复联 / 其他
3. 你的称呼：（聊天中对方怎么叫你）
4. 对方称呼：（你怎么叫对方）
5. 时长：认识多久 / 在一起多久
```

## 第 2 轮：典型场景

```
能否描述一个最近的互动场景？比如：
- 一次让你开心的互动
- 一次让你困惑或难受的互动
- 一次冲突或冷战
```

## 第 3 轮：数据来源

```
你能提供什么数据？
- 直接粘贴聊天记录
- 导出文件（CSV / TXT / JSONL / SQLite）
- 截图
- CLI 预处理产物（digest.md + evidence.jsonl）
```

## 确认汇总

收集完毕后展示：

```
信息汇总：
  关系类型：{type}
  当前状态：{status}
  你的称呼：{self_name}
  对方称呼：{target_name}
  时长：{duration}
  数据来源：{source}
  典型场景：{scene_summary}

确认？（确认 / 修改 [字段名]）
```

## 跳过规则

- 第 1 轮跳过：关系类型默认"未指定"，其余留空
- 第 2 轮跳过：标注"无典型场景，分析时自行识别"
- 第 3 轮跳过：提示用户何时可以提供数据
