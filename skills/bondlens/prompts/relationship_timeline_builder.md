# BondLens Relationship Timeline Builder

正式判断前先生成关系时间线。时间线不是流水账，而是把 T1/T2 信号组织成关系阶段，防止用局部证据覆盖全局变化。

## 输入

- `relationship_signal_ledger.jsonl`
- `contradiction_ledger.md`
- 用户 intake 和用户纠正

## 输出

```markdown
# 关系时间线

## 阶段 1：{阶段名}
**时间**: {start} 至 {end}
**主导信号**: 靠近|拒绝|冷淡|修复|重新定义关系|开放条件
**关键证据**: {IDs}
**阶段判断**: {1-2 句}
**反证/未解释点**: {IDs 或说明}

## 阶段 2：...

## 当前阶段
**最近 T1/T2**: {IDs}
**当前可判断**: {能判断什么}
**当前不能判断**: {不能判断什么}
```

## 阶段划分规则

1. 阶段边界只由 T1/T2 触发，不由消息量变化触发。
2. 同一阶段内可以有矛盾信号，必须标注为"摇摆/并存"，不要强行消解。
3. 早期拒绝不能直接覆盖后期条件性接受；后期暧昧也不能抹掉早期边界。
4. 行动卡只锚定当前阶段和最近 1 个月，但必须知道它从哪个历史阶段演变而来。

## 当前阶段判断

当前阶段必须输出三句话：

1. 数据足够支持的判断。
2. 数据仍不能支持的判断。
3. 本周行动应避开的最大误判。

示例：

> 能判断：对方没有关闭关系讨论，但当前无法承接明确推进。
> 不能判断：不能说对方已经准备好进入关系。
> 最大误判：把"仍愿意聊"误读为"可以继续追问答案"。
