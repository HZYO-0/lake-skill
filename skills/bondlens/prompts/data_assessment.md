# BondLens Data Assessment Prompt

评估数据充分性，决定分析深度。

## 评估维度

| 维度 | 不足 (< 30 条) | 基本 (30-100 条) | 充分 (100+ 条) |
|------|---------------|-----------------|---------------|
| 消息量 | 仅输出局部观察 | 分析但标注低置信度 | 完整分析 |
| 时间跨度 | < 1 天：单场景快照 | 1-7 天：短期模式 | 1+ 周：趋势分析 |
| 双方参与 | 只有一方：无法分析互动 | 严重不平衡：标注偏差 | 基本均衡：可信互动模式 |
| 场景多样性 | 只有闲聊：仅分析日常模式 | 有冲突：可分析冲突路径 | 冲突+修复：完整互动循环 |

## 关系信号覆盖

消息量充分不等于关系判断充分。必须额外检查 T1/T2 覆盖：

| 维度 | 不足 | 基本 | 充分 |
|------|------|------|------|
| T1 关系定义 | 没有表白/拒绝/关系定义/边界/自我认知 | 有 1 类 T1，但缺少回应或后续 | 有 2+ 类 T1，且包含回应和后续 |
| T1 完整性 | 只有片段，缺上下文 | 有原话但缺后续变化 | 有原话、即时语境、后续变化 |
| T2 转折 | 没有冲突/修复/冷淡/靠近 | 有转折但时间点不清 | 有转折、触发、修复或后果 |
| 用户纠正 | 未收集 | 有纠正但未入台账 | 纠正已进入 `relationship_signal_ledger` |

## 决策矩阵

```
if 消息量 < 30:
    → 输出：局部观察 + 提示补充数据
    → 不输出：人格画像、依恋假设、互动模式

elif 消息量 30-100:
    → 输出：初步人格信号（标注低置信度）、简单互动模式
    → 标注：哪些维度因数据不足无法分析

elif 消息量 100+:
    if T1 关系定义覆盖不足:
        → 输出：背景分析 + 数据缺口 + 暂不输出强关系判断
        → 行动卡降级为"补充/核对关键语境"
    else:
        → 输出：完整 9 层分析（Layer -1 行动卡 + Layer 0-7）
        → 每个结论标注置信度
```

## 强结论门槛

以下结论必须有 T1/T2 支撑，否则禁止输出：

- "对方喜欢/不喜欢"
- "对方接受/拒绝发展"
- "当前应该推进/放弃"
- "对方呈现回避/焦虑相关信号"
- "某个行为是关系雷区"

如果只有 T4 统计或日常闲聊，改写为：

> 当前数据能描述日常互动风格，但不足以判断关系性质。建议补充关系定义、边界、冲突或修复相关对话。

## 数据质量问题

需额外标注：
- 时间戳缺失或不连续
- 一方消息占比 > 80%
- 全部为同一场景（如只有工作讨论）
- 明显的转发/复制内容
- T1 关系定义对话缺失或只有单方转述
- 用户纠正尚未写入台账
