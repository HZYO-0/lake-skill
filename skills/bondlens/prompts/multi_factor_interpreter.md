# BondLens Multi-Factor Interpreter

关系结论不得压成单因子判断。核心问题必须用多因子模型解释：不是"喜欢/不喜欢"二选一，而是多个力量同时作用。

## 必选因子

每个核心关系判断至少评估以下因子：

| 因子 | 说明 |
|------|------|
| affection_attraction | 是否存在好感、靠近、未来想象、亲密表达 |
| avoidance_self_protection | 是否存在回避、退缩、害怕关系暴露、自我保护 |
| reality_constraints | 距离、毕业、工作、实验室、身份关系等现实限制 |
| guilt_indebtedness | 愧疚、亏欠、被照顾后的压力 |
| self_worth | "不配"、拖累、耽误对方、自我贬低 |
| safety_trust | 是否信任、是否能承受冲突、是否担心被控制 |
| user_pressure | 用户追问、确认需求、控制感、节奏压力 |

## 输出格式

```markdown
### {核心判断}

**多因子解释**:
- 感情/吸引: {强|中|弱}，证据：{IDs}，反证：{IDs}
- 回避/自我保护: {强|中|弱}，证据：{IDs}，反证：{IDs}
- 现实顾虑: {强|中|弱}，证据：{IDs}，反证：{IDs}
- 愧疚/自我价值: {强|中|弱}，证据：{IDs}，反证：{IDs}
- 用户压力: {强|中|弱}，证据：{IDs}，反证：{IDs}

**合成结论**:
{A+B+C 如何共同导致当前行为。不要写成单一原因。}

**不能推出的结论**:
- 不能仅凭 {证据} 推出 {过度结论}
- 还需要观察 {缺失信号}
```

## 解释约束

1. 出现 T1 冲突时，优先输出"多因子并存"，不要在两端横跳。
2. "拒绝"和"有感情"可以同时存在；必须解释为什么没有转化为接受。
3. "回避信号"和"现实顾虑"可以同时存在；不要互相排除。
4. "用户压力"只描述行为影响，不责备用户。
5. 如果 T1 缺失，输出"不足以判断关系性质"，不补脑。

## 禁止表达

- "她就是不喜欢你"
- "她就是回避型"
- "完全没可能"
- "只是在找借口"
- "所有行为都说明……"

替代写法：

> 台账显示至少有三组力量同时存在：{A}、{B}、{C}。因此当前更稳妥的判断是 {阶段性结论}，置信度为 {level}。
