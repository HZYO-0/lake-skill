# 修正处理 Prompt

## 任务

识别用户的纠正意图，生成修正记录。

---

## 触发条件

以下表达视为纠正指令：
- "他不会这样" / "他不会这么说"
- "你理解错了" / "这不对"
- "他应该是" / "他其实是" / "他更倾向于"
- "你说的不像他" / "感觉不太像"
- "他遇到这种情况会..."
- "他其实..."

---

## 处理步骤

### Step 1: 理解纠正内容

从用户的话中提取：
- **场景**：在什么情况下发生
- **错误行为**：AI 做了什么不像 TA 的判断
- **正确行为**：TA 实际上会怎么做

如果用户说得模糊，追问一次：
```
我理解了，他在 [场景] 的时候应该 [正确行为]，对吗？
```

### Step 2: 判断归属

- 涉及对方的行为模式 → 归到 **对方画像**（Layer 2）
- 涉及用户自己的行为 → 归到 **我的画像**（Layer 3）
- 涉及互动模式 → 归到 **互动模式**（Layer 4）
- 涉及依恋信号 → 归到 **依恋信号**（Layer 5）

### Step 3: 生成 Correction 记录

输出 JSON 格式：

```json
{
  "scene": "当 TA 收到我的长消息时",
  "wrong": "AI 认为 TA 会冷淡回复",
  "correct": "TA 其实会认真看完再回复，只是回复比较慢",
  "layer": "layer_2",
  "timestamp": "2026-06-03"
}
```

多条修正：
```json
{
  "corrections": [
    {"scene": "...", "wrong": "...", "correct": "...", "layer": "..."},
    {"scene": "...", "wrong": "...", "correct": "...", "layer": "..."}
  ]
}
```

### Step 4: 检查冲突

如果新修正与已有分析冲突：
```
这条纠正与已有分析冲突：
- 已有分析：{已有描述}（证据：{IDs}）
- 你的纠正：{新描述}

以你的纠正为准，更新已有分析？还是两者都保留（适用于不同场景）？
```

### Step 5: 确认并写入

确认修正内容后，更新对应 Layer 的内容，并在 Correction 记录中追加。

---

## Correction 记录格式

在报告末尾的 Correction 记录中追加：

```markdown
## Correction 记录

### 2026-06-03
- 场景：{scene}
- 原判断：{wrong}
- 修正为：{correct}
- 归属：{layer}
```

---

## 维护规则

- 每个报告最多保留 50 条 correction
- 超出时，将语义相近的 correction 合并为 1 条
- 合并时优先保留最新的表述
- 每次合并告知用户："已将 {N} 条相似规则合并为 {M} 条"
