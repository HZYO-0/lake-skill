# 增量合并 Prompt

## 任务

将新数据合并到已有分析报告中。

**原则：只追加增量，不覆盖已有结论。如有冲突，输出冲突提示让用户决定。**

---

## Step 1: 分类

将新信息归入对应维度：

| 信息类型 | 归入 |
|---------|------|
| 对方表达习惯、口头禅、情绪模式 | → Layer 2 对方表达 DNA |
| 用户自己的表达习惯 | → Layer 3 我的表达 DNA |
| 互动循环、冲突模式、修复信号 | → Layer 4 互动模式 |
| 依恋信号、安全感信号 | → Layer 5 依恋信号 |
| 沟通建议、消息草稿 | → Layer 6 沟通建议 |
| 多个维度 | → 分别归入 |

---

## Step 2: 冲突检测

对比新内容与已有分析：

- 如果新内容**补充**了已有信息（增加了新细节）→ 直接追加
- 如果新内容**确认**了已有信息 → 忽略（不重复写）
- 如果新内容**与已有信息矛盾** → 输出冲突提示：

```
发现冲突：
- 已有：{已有描述}
- 新发现：{新内容描述}
- 来源：{证据 ID / 用户反馈}

建议：[保留已有 / 更新为新内容 / 两者都保留并标注时间]
请决定。
```

---

## Step 3: 生成 Patch

输出格式：

```
=== 对方表达 DNA 更新 ===

[追加到"Layer 2/口头禅"节]
- 新口头禅："{xxx}"（证据：{IDs}）

[追加到"Layer 2/情绪触发器"节]
- {新观察}（证据：{IDs}）

[无更新] 或 [以上章节有更新]
```

```
=== 互动模式更新 ===

[追加到"Layer 4/负向循环"节]
- {新模式}（证据：{IDs}）

[无更新] 或 [以上章节有更新]
```

---

## Step 4: 更新摘要

```
本次更新摘要：
- 对方表达 DNA：追加 {N} 条新信息
- 互动模式：追加 {N} 条新信息
- 依恋信号：追加 {N} 条新信息
- 发现 {N} 处冲突，需要你确认（见上方）

确认应用更新？
```

---

## 版本管理

每次更新自动：
1. 备份当前报告到 `versions/v{N}/`
2. 版本号 +1
3. 记录更新日志
