# 数据充分性评估 Prompt

## 任务

在分析前评估输入数据是否足够支撑完整分析。

---

## 评估维度

### 1. 消息量

| 数量 | 评估 |
|------|------|
| < 30 条 | 不足：仅输出低置信局部观察 |
| 30-100 条 | 基本：可输出初步分析，标注低置信 |
| 100-500 条 | 良好：可输出完整分析，中等置信 |
| 500+ 条 | 充分：可输出完整分析，置信度合理 |

### 2. 时间跨度

| 跨度 | 评估 |
|------|------|
| < 1 天 | 不足：无法观察模式 |
| 1-7 天 | 基本：仅限短期模式 |
| 1-4 周 | 良好：可观察中期趋势 |
| 1+ 月 | 充分：可观察长期模式 |

### 3. 双方参与

| 情况 | 评估 |
|------|------|
| 只有一方消息 | 不足：无法分析互动模式 |
| 双方都有，但严重不平衡 | 基本：互动分析受限 |
| 双方基本均衡 | 良好：可分析互动 |

### 4. 场景多样性

| 情况 | 评估 |
|------|------|
| 只有日常闲聊 | 基本：缺乏冲突/修复数据 |
| 包含冲突场景 | 良好：可分析冲突模式 |
| 包含冲突 + 修复 | 充分：可分析完整循环 |
| 包含多种场景 | 充分：可分析多维度模式 |

### 5. 数据质量

| 情况 | 评估 |
|------|------|
| ASR/OCR 置信度低 | 降低置信度 |
| 时间戳缺失/不准确 | 影响响应时间分析 |
| 系统消息混入 | 需过滤 |

---

## 输出

```
数据评估：
- 消息量：{count} 条 → {评估}
- 时间跨度：{span} → {评估}
- 双方参与：{balance} → {评估}
- 场景多样性：{diversity} → {评估}
- 数据质量：{quality} → {评估}

综合评估：{充分 / 基本 / 不足}

建议：
{如果不足：建议补充哪些数据}
{如果基本：哪些分析维度受限}
{如果充分：可以进行完整分析}
```

---

## 决策规则

- **充分** → 进入 Step 3 完整分析
- **基本** → 进入 Step 3，但标注低置信维度
- **不足** → 仅输出局部观察，提示用户补充数据
