# 互动模式分析 Prompt

## 任务

分析双方的互动循环模式。参考 `references/frameworks/relationship_communication_patterns.md`。

BondLens 的核心价值：不只分析一个人，而是分析**两个人之间的互动**。

---

## 正向循环

识别让关系向好的互动模式：

**模式模板**：
```
{谁} 做了 {什么} → {对方} 回应 {什么} → {谁} 再 {什么} → 关系向好
```

**常见正向循环**：
- 分享日常 → 回应关心 → 继续分享 → 信任增加
- 承认情绪 → 软化回应 → 共同修复 → 亲密度提升
- 给空间 → 主动联系 → 表达想念 → 关系升温
- 表达需求 → 倾听理解 → 协商妥协 → 满足感提升

```
正向循环 1：[描述]（证据：{IDs}，置信度：{confidence}）
正向循环 2：[描述]（证据：{IDs}，置信度：{confidence}）
```

---

## 负向循环

识别让关系变差的互动模式：

**常见负向循环**：
- 施压 → 退缩 → 更大施压 → 更远退缩
- 模糊信号 → 过度解读 → 焦虑表达 → 对方厌烦
- 沉默测试 → 对方不安 → 追问 → 更沉默
- 间接表达 → 对方没理解 → 指责 → 防御

```
负向循环 1：[描述]（证据：{IDs}，置信度：{confidence}）
负向循环 2：[描述]（证据：{IDs}，置信度：{confidence}）
```

---

## 冲突升级路径

识别冲突如何升级：

**升级信号**（参考 relationship_communication_patterns.md）：
- 反复质问
- 讽刺或间接指责
- 沉默后的长篇情绪独白
- 用冷淡测试
- 要求立即确认

**升级路径模板**：
```
触发事件 → {谁} 的反应 → {对方} 的反应 → 升级 → 更严重后果
```

```
升级路径 1：[描述]（证据：{IDs}）
升级路径 2：[描述]（证据：{IDs}）
```

---

## 修复信号

识别冲突后如何修复：

**修复信号**（参考 relationship_communication_patterns.md）：
- 带有具体责任的道歉
- 不带指责的重新表述
- 给空间但约定后续时间
- 清晰表达需求
- 先确认再提问

**修复模式**：
- 谁更常主动修复？
- 修复需要多长时间？
- 修复方式有效吗？

```
修复模式：[描述]（证据：{IDs}）
修复时间：[通常多久]
修复效果：[有效/部分有效/无效]
```
