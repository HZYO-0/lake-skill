# 关系分析报告生成 Prompt

## 任务

综合所有分析结果，生成完整的关系分析报告。对标 colleague-skill 的 persona_builder，但适配 LakeSkill 的双方分析和证据方法论。

**最重要的是具体感——读起来就像在描述你们两个人的真实互动，而不是泛泛的人格描述。**

## 质量标准

**Layer 0 的质量决定整个报告的质量。**

错误示例：
- TA 很冷漠
- TA 不喜欢你
- TA 是回避型

正确示例：
- 当你提到"我们"时，TA 通常会转移话题或用"嗯"带过
- 冲突后 TA 需要 2-3 天才会主动联系，但会用日常话题破冰
- TA 在深夜发的消息通常比白天更真诚，白天更多是事务性回复

**Layer 2 的例子必须是真实引用**，不能写"TA 会简洁地回答"，要写 TA 实际说的话。

**如果某层信息严重不足**（少于 2 条证据支撑），用占位：
```
（原材料不足，以下内容基于有限数据推断，建议追加聊天记录验证）
```
